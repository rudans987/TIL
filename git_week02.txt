협업하는 단계
1단계 : 누가 이 작업을 할것인지 정한다-issue
2단계 : 각자 맡은 것을 작업한다.-branch
3단계 : 각자 작업을 프로젝트에 합친다.-merge
경우에따라 작업한 내용을 리뷰하고 반영 후 합친다.-pr 후 merge

2.
issue : 내가할 작업, 기능 추가, 버그 리포트 등 여러방식으로 사용
 branch : 특정 커밋에서 갈라져나와 작업할때 뻗어나오는 가지
merge : 작업 내역들을 다른 branch로 합치는 것
merge conflict : 서로 같은곳을 다르게 수정하고 머지할때 발생
사람이 직접 보고 수정해야함
tracking : 로컬 repo와 원격 repo를 연결해주는 것
